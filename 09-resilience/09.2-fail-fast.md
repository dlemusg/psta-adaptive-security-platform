# Fail Fast Strategy

## Propósito

Este documento describe la estrategia de Fail Fast para la Plataforma de Seguridad Transaccional Adaptativa (PSTA).

El objetivo es detectar errores lo más temprano posible, evitar consumo innecesario de recursos y prevenir fallas en cascada.

---

# Principios

## Fallar Rápido

Cuando una dependencia no puede responder dentro del tiempo esperado, la solicitud debe finalizar inmediatamente.

```text
Esperar indefinidamente = Propagar la falla
```

---

## No Ocultar Errores

Los errores deben ser explícitos y trazables.

```text
Timeout
Validación Inválida
Servicio No Disponible
Configuración Incorrecta
```

---

## Proteger Recursos

Fail Fast evita:

```text
Conexiones agotadas
Memoria excesiva
Saturación de Pods
Latencia acumulada
```

---

# Aplicación en la Plataforma

## Risk Assessment

Si el motor de riesgo no responde:

```text
RiskAssessmentUnavailable
```

Acción:

```text
Incrementar Riesgo
Solicitar Step-Up Authentication
```

---

## Adaptive Authentication

Si el proveedor OTP no responde:

```text
OTPProviderUnavailable
```

Acción:

```text
Activar Fallback Authentication
```

---

## Redis

Si no es posible realizar velocity checks:

```text
VelocityCheckUnavailable
```

Acción:

```text
Asumir Riesgo Mayor
```

---

## Core Bancario

Si el Core no responde:

```text
TransactionExecutionUnavailable
```

Acción:

```text
No aprobar transacciones de forma automática
```

---

# Integración con WebFlux

Las operaciones deben utilizar:

```text
Timeouts
Validaciones Tempranas
Manejo Reactivo de Errores
```

Ejemplo conceptual:

```java
riskClient.evaluate(transaction)
    .timeout(Duration.ofMillis(120))
    .onErrorResume(...)
```

---

# Conclusión

La estrategia Fail Fast permite que la plataforma detecte y gestione errores de forma temprana, evitando degradaciones progresivas y habilitando mecanismos de resiliencia como circuit breakers y fallbacks. En una plataforma antifraude en tiempo real, es preferible fallar rápidamente y tomar decisiones controladas que operar con información incompleta o respuestas tardías.
