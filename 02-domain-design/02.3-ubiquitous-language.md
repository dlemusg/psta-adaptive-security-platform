# Lenguaje Ubicuo

## Propósito

Este documento define el Lenguaje Ubicuo (Ubiquitous Language) utilizado en la Plataforma de Seguridad Transaccional Adaptativa (PSTA).

El objetivo es establecer una terminología común entre negocio, arquitectura, desarrollo, seguridad, fraude y operaciones, reduciendo ambigüedades y garantizando una comunicación consistente entre todos los equipos involucrados.

---

## Principios

El lenguaje ubicuo debe cumplir las siguientes reglas:

- Utilizar términos del negocio y no de la tecnología.
- Mantener una única definición para cada concepto.
- Ser utilizado en documentación, código, eventos y conversaciones.
- Evitar sinónimos para el mismo concepto.
- Evolucionar junto con el dominio.

---

## Términos Generales

| Término | Definición |
|----------|------------|
| Cliente | Persona titular de una cuenta o producto financiero. |
| Sesión | Interacción autenticada entre un cliente y la plataforma. |
| Canal | Medio por el cual se origina una operación (Web, Móvil, B2B). |
| Operación | Acción realizada por un cliente que requiere validación y autorización. |
| Transacción | Operación financiera susceptible de evaluación de riesgo. |
| Beneficiario | Destinatario de una transferencia o movimiento financiero. |

---

# Dominio de Riesgo y Fraude

| Término | Definición |
|----------|------------|
| Riesgo | Probabilidad de que una operación represente una amenaza para la organización o el cliente. |
| Evaluación de Riesgo | Proceso mediante el cual se calcula el nivel de riesgo asociado a una operación o sesión. |
| Risk Score | Valor numérico que representa el riesgo calculado. |
| Nivel de Riesgo | Clasificación categórica del score (Bajo, Medio, Alto, Crítico). |
| Señal de Riesgo | Indicador utilizado para construir la evaluación de riesgo. |
| Patrón de Fraude | Comportamiento identificado como potencialmente fraudulento. |
| Fraude por Velocidad | Patrón caracterizado por múltiples transacciones realizadas en un corto período de tiempo. |
| Correlación de Eventos | Capacidad de relacionar múltiples eventos para detectar comportamientos anómalos. |
| Umbral de Riesgo | Valor a partir del cual se ejecutan controles adicionales. |
| Riesgo Crítico | Nivel de riesgo que requiere acciones inmediatas de mitigación. |

# Dominio de Autenticación Adaptativa

| Término | Definición |
|----------|------------|
| Autenticación Adaptativa | Selección dinámica de mecanismos de autenticación según el nivel de riesgo. |
| Factor de Autenticación | Método utilizado para validar la identidad del cliente. |
| Desafío de Autenticación | Solicitud de validación enviada al cliente. |
| Step-Up Authentication | Incremento del nivel de autenticación requerido debido al riesgo. |
| Push Authentication | Autenticación mediante aprobación desde una aplicación móvil. |
| OTP (One-Time Password) | Clave dinámica de un solo uso. |
| Biometría | Validación basada en características biométricas del usuario. |
| Fallback | Mecanismo alternativo utilizado cuando un factor principal no está disponible. |
| Resultado de Autenticación | Estado final de un desafío (Éxito o Fallo). |
| Sesión Confiable | Sesión con un nivel de confianza suficiente para reducir desafíos adicionales. |

---

# Dominio de Gestión de Dispositivos

| Término | Definición |
|----------|------------|
| Dispositivo | Equipo utilizado por el cliente para interactuar con la plataforma. |
| Device Fingerprint | Identificador único generado a partir de características del dispositivo. |
| Dispositivo Confiable | Dispositivo previamente validado y autorizado. |
| Dispositivo Desconocido | Dispositivo que no posee historial de confianza. |
| Device Trust | Nivel de confianza asociado a un dispositivo. |
| Device Registration | Proceso de registro de un nuevo dispositivo. |
| Device Reputation | Historial de comportamiento asociado al dispositivo. |

---

# Dominio de Protección de Cuenta

| Término | Definición |
|----------|------------|
| Bloqueo de Cuenta | Acción preventiva que impide temporalmente el uso de una cuenta. |
| Restricción de Cuenta | Limitación parcial de funcionalidades debido a riesgo. |
| Revisión Manual | Evaluación realizada por un analista humano. |
| Mitigación | Acción aplicada para reducir el riesgo identificado. |
| Escalamiento | Transferencia de un caso a un proceso de análisis especializado. |

---

# Dominio de Decisión Transaccional

| Término | Definición |
|----------|------------|
| Decisión Transaccional | Resultado final de la evaluación de una operación. |
| Operación Aprobada | Transacción autorizada para continuar su procesamiento. |
| Operación Rechazada | Transacción que no cumple los criterios de seguridad. |
| Operación Restringida | Transacción condicionada por controles adicionales. |
| Flujo Transaccional | Secuencia completa desde la solicitud hasta la decisión final. |

---

# Eventos de Dominio

| Término | Definición |
|----------|------------|
| Evento de Dominio | Hecho relevante ocurrido dentro de un contexto de negocio. |
| Evento Publicado | Evento emitido por un contexto. |
| Evento Consumido | Evento procesado por un contexto. |
| CorrelationId | Identificador que permite rastrear una operación a través de múltiples servicios. |
| Event Version | Versión del contrato de un evento. |
| Event Producer | Contexto que genera un evento. |
| Event Consumer | Contexto que consume un evento. |

# Observabilidad y Operación

| Término | Definición |
|----------|------------|
| TraceId | Identificador técnico utilizado para trazabilidad distribuida. |
| Latencia | Tiempo requerido para completar una operación. |
| SLA | Acuerdo de nivel de servicio comprometido. |
| SLO | Objetivo interno de nivel de servicio. |
| Disponibilidad | Porcentaje de tiempo en que la plataforma permanece operativa. |
| Incidente | Evento que afecta la operación normal del sistema. |
| Runbook | Procedimiento documentado para atender incidentes. |
| Dashboard | Vista consolidada de métricas operativas y de negocio. |

# Seguridad

| Término | Definición |
|----------|------------|
| Zero Trust | Modelo donde ninguna entidad es confiable por defecto. |
| mTLS | Autenticación mutua mediante certificados digitales. |
| JWT de Servicio | Token utilizado para autenticación entre servicios. |
| Cifrado en Tránsito | Protección de datos durante la comunicación. |
| Cifrado en Reposo | Protección de datos almacenados. |
| Secreto | Credencial o dato sensible utilizado por la plataforma. |
| Vault | Sistema de gestión centralizada de secretos. |
| Auditoría | Registro de evidencia para cumplimiento y trazabilidad. |

---

# Niveles de Riesgo

La plataforma utilizará una clasificación estándar para la toma de decisiones.

| Nivel | Descripción |
|---------|---------|
| Bajo | Operación consistente con el comportamiento esperado. |
| Medio | Operación con señales moderadas de riesgo. |
| Alto | Operación con múltiples señales de riesgo relevantes. |
| Crítico | Operación con alta probabilidad de fraude o compromiso. |

---

# Acciones de Mitigación

| Acción | Descripción |
|----------|------------|
| Aprobar | Permitir la ejecución de la operación. |
| Solicitar Autenticación | Requerir validación adicional. |
| Incrementar Autenticación | Aplicar Step-Up Authentication. |
| Restringir | Limitar temporalmente capacidades de la cuenta. |
| Bloquear | Impedir la ejecución de operaciones. |
| Escalar | Transferir el caso a revisión especializada. |

---

# Regla Fundamental

Todo término utilizado en:

- Diagramas.
- Eventos.
- APIs.
- Código.
- Documentación.
- Conversaciones de negocio.

Se debe utilizar las definiciones establecidas en este documento para garantizar consistencia semántica en toda la plataforma.
