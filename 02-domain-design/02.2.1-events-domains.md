# Catálogo de Eventos de Dominio

## Decisión Transaccional

### Eventos Publicados

| Evento | Descripción | Consumidores | Momento de Publicación | Payload - Ejemplo |
|----------|------------|------------|------------|------------|
| TransactionReceived | Se recibe una nueva solicitud transaccional y se inicia el flujo de evaluación de seguridad. | Monitoreo de Riesgo, Auditoría | Al ingresar una nueva transacción desde cualquier canal. | 
| TransactionApproved | La operación fue aprobada exitosamente después de completar las validaciones requeridas. | Perfil de Cliente, Auditoría, Analítica | Después de una evaluación y autenticación exitosas. |
| TransactionRejected | La operación fue rechazada por riesgo, autenticación fallida o políticas de seguridad. | Monitoreo de Riesgo, Auditoría, Perfil de Cliente | Cuando no es posible aprobar la operación. |
| TransactionRequiresAuthentication | La operación requiere autenticación adicional debido al nivel de riesgo calculado. | Autenticación Adaptativa | Cuando el riesgo supera el umbral permitido para aprobación silenciosa. |



### Eventos Consumidos 

| Evento | Productor | Descripción |
|----------|------------|------------|
| RiskScoreCalculated | Monitoreo de Riesgo | Resultado de la evaluación de riesgo asociada a la transacción. |
| AuthenticationCompleted | Autenticación Adaptativa | Indica que el cliente completó exitosamente el desafío de autenticación. |
| AuthenticationFailed | Autenticación Adaptativa | Indica que el desafío de autenticación falló o expiró. |
| AccountBlocked | Protección de Cuenta | Notifica que la cuenta fue bloqueada preventivamente. |

## Monitoreo de Riesgo y Fraude

### Eventos Publicados

| Evento | Descripción | Consumidores | Momento de Publicación |
|----------|------------|------------|------------|
| RiskEvaluationRequested | Solicita la evaluación de riesgo para una transacción o sesión. | Motor de Riesgo, Auditoría | Al iniciar una evaluación de riesgo. |
| RiskScoreCalculated | Resultado oficial de la evaluación de riesgo. | Decisión Transaccional, Autenticación Adaptativa, Auditoría | Al finalizar el cálculo de riesgo. |
| FraudPatternDetected | Se detecta un patrón sospechoso mediante correlación de eventos. | Protección de Cuenta, Auditoría | Cuando una regla o correlación identifica un posible fraude. |
| CriticalRiskDetected | Se identifica un riesgo crítico que requiere acciones inmediatas. | Protección de Cuenta, Autenticación Adaptativa, Auditoría | Cuando el score supera el umbral crítico definido. |

### Eventos Consumidos

| Evento | Productor | Descripción |
|----------|------------|------------|
| TransactionReceived | Decisión Transaccional | Inicio de evaluación de riesgo para una nueva transacción. |
| AuthenticationCompleted | Autenticación Adaptativa | Actualiza el contexto de confianza del cliente. |
| DeviceTrustGranted | Gestión de Dispositivos | Actualiza señales de riesgo asociadas al dispositivo. |
| DeviceTrustRevoked | Gestión de Dispositivos | Incrementa el riesgo asociado a la sesión. |

## Autenticación Adaptativa

### Eventos Publicados

| Evento | Descripción | Consumidores | Momento de Publicación |
|----------|------------|------------|------------|
| AuthenticationRequested | Solicita la ejecución de un mecanismo de autenticación. | Proveedores de Factores, Auditoría | Cuando una operación requiere autenticación. |
| AuthenticationCompleted | El cliente completó exitosamente el desafío solicitado. | Decisión Transaccional, Riesgo, Perfil de Cliente | Después de una autenticación exitosa. |
| AuthenticationFailed | El cliente falló o abandonó el desafío de autenticación. | Decisión Transaccional, Riesgo, Auditoría | Después de un intento fallido. |
| SessionTrustUpdated | Se actualiza el nivel de confianza de la sesión. | Riesgo, Auditoría | Cuando cambia el contexto de confianza. |

### Eventos Consumidos

| Evento | Productor | Descripción |
|----------|------------|------------|
| RiskScoreCalculated | Monitoreo de Riesgo | Determina el factor de autenticación requerido. |
| CriticalRiskDetected | Monitoreo de Riesgo | Obliga autenticación reforzada o rechazo. |
| TransactionRequiresAuthentication | Decisión Transaccional | Solicita autenticación adicional para continuar el flujo. |

## Protección de Cuenta

### Eventos Publicados

| Evento | Descripción | Consumidores | Momento de Publicación |
|----------|------------|------------|------------|
| AccountBlocked | Se bloquea preventivamente una cuenta. | Decisión Transaccional, Riesgo, Auditoría, Notificaciones | Cuando se detecta riesgo crítico o fraude confirmado. |
| AccountUnblocked | Se levanta una restricción previamente aplicada. | Riesgo, Auditoría | Tras validación o revisión manual. |
| AccountRestricted | Se aplican restricciones parciales a la cuenta. | Riesgo, Auditoría | Cuando se requiere mitigación temporal. |

### Eventos Consumidos

| Evento | Productor | Descripción |
|----------|------------|------------|
| CriticalRiskDetected | Monitoreo de Riesgo | Dispara acciones de protección automática. |
| FraudPatternDetected | Monitoreo de Riesgo | Inicia evaluación de bloqueo o restricción. |

## Eventos Críticos para Correlación

Los siguientes eventos son considerados críticos para el monitoreo de fraude y deben tener prioridad de procesamiento:

| Evento | Prioridad |
|----------|----------|
| TransactionReceived | Alta |
| RiskScoreCalculated | Alta |
| FraudPatternDetected | Crítica |
| CriticalRiskDetected | Crítica |
| AuthenticationFailed | Alta |
| AccountBlocked | Crítica |
| TransactionApproved | Alta |

## Estructura Base de Eventos

Todos los eventos de dominio deben cumplir una estructura estándar para garantizar trazabilidad, auditoría y evolución de contratos.

| Campo | Tipo | Descripción |
|----------|----------|------------|
| eventId | UUID | Identificador único del evento. |
| eventType | String | Nombre del evento de dominio. |
| correlationId | String | Identificador de correlación de negocio. |
| transactionId | String | Identificador de la transacción asociada. |
| producer | String | Contexto que genera el evento. |
| timestamp | Timestamp | Fecha y hora de generación. |
| version | String | Versión del contrato del evento. |

## Convenciones

- Los eventos son inmutables.
- Los contratos son versionados.
- Los consumidores deben ser tolerantes a nuevos campos.
- Ningún evento puede contener información sensible en texto plano.
- Todos los eventos deben ser auditables y trazables mediante `correlationId`.
