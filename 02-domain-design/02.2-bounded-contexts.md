# Bounded Contexts

## Propósito

Este documento define los Bounded Contexts identificados para la Plataforma de Seguridad Transaccional Adaptativa (PSTA) utilizando los principios de Domain-Driven Design (DDD).

El objetivo es establecer límites claros de responsabilidad, lenguaje ubicuo y ownership para cada dominio, reduciendo acoplamiento y facilitando la evolución independiente de las capacidades de negocio.

---

# Visión General

La plataforma se organiza alrededor de los siguientes dominios:

| Dominio | Tipo |
|----------|----------|
| Decisión Transaccional | Core Domain |
| Monitoreo de Riesgo y Fraude | Core Domain |
| Autenticación Adaptativa | Core Domain |
| Gestión de Dispositivos | Supporting Domain |
| Perfil de Cliente | Supporting Domain |
| Protección de Cuenta | Supporting Domain |
| Proveedores de Factores | Supporting Domain |
| Auditoría y Evidencia | Generic Domain |

---

# 1. Decisión Transaccional

## Tipo

Core Domain

## Responsabilidad

Orquestar la evaluación de riesgo, la autenticación adaptativa y la decisión final sobre una operación.

Este dominio representa el punto central de coordinación entre los dominios críticos de seguridad.

---

## Capacidades

- Recibir solicitudes transaccionales.
- Solicitar evaluación de riesgo.
- Solicitar autenticación cuando sea requerida.
- Aprobar operaciones.
- Rechazar operaciones.
- Escalar bloqueos.

---

## Lenguaje Ubicuo

| Término | Definición |
|----------|------------|
| Transaction | Operación iniciada por un cliente |
| Decision | Resultado final de una evaluación |
| Approval | Operación aprobada |
| Rejection | Operación rechazada |
| Step-Up Authentication | Incremento del nivel de autenticación requerido |

---

## Eventos Publicados

- TransactionReceived
- TransactionApproved
- TransactionRejected
- TransactionRequiresAuthentication

---

## Eventos Consumidos

- RiskScoreCalculated
- AuthenticationCompleted
- AuthenticationFailed
- AccountBlocked

---

# Principio de Diseño

Cada Bounded Context posee:

- Su propio modelo de dominio.
- Su propio lenguaje ubicuo.
- Su propio ciclo de vida.
- Su propia capacidad de despliegue.
- Su propio ownership.

La colaboración entre contextos se realiza mediante contratos explícitos y eventos de negocio, evitando dependencias directas sobre modelos internos de otros dominios.
