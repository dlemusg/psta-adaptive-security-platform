# Estrategia de Pruebas

## Propósito

Este documento describe la estrategia de pruebas adoptada para la Plataforma de Seguridad Transaccional Adaptativa (PSTA).

La estrategia está alineada con:

- Clean Architecture.
- Scaffold Bancolombia.
- Domain Driven Design (DDD).
- Event-Driven Architecture.
- DevSecOps.
- Continuous Testing.

El objetivo es garantizar la calidad funcional, técnica y operacional de una plataforma antifraude crítica para el negocio.

---

# Objetivos

La estrategia de pruebas debe permitir:

- Validar reglas de negocio.
- Detectar defectos tempranamente.
- Reducir regresiones.
- Garantizar compatibilidad entre servicios.
- Validar resiliencia operacional.
- Validar escenarios de fraude.
- Garantizar estabilidad en producción.

---

# Estrategia General

La solución adopta una estrategia de pruebas multinivel alineada con la arquitectura limpia.

```text
                    Acceptance Tests
                 (Business Scenarios)

                           ▲

                     End-to-End Tests

                           ▲

                     Contract Tests

                           ▲

                   Integration Tests

                           ▲

                    Component Tests

                           ▲

                        Unit Tests
```

Cada nivel incrementa el alcance validado y reduce el nivel de aislamiento.

---

# Relación con Clean Architecture

| Nivel de Prueba | Capa Principal |
|-----------------|---------------|
| Unit Tests | Domain Model + Use Cases |
| Component Tests | Entry Points + Use Cases |
| Integration Tests | Driven Adapters |
| Contract Tests | APIs REST + Eventos Kafka |
| Acceptance Tests | Escenarios de Negocio |
| End-to-End Tests | Plataforma Completa |

La estrategia de pruebas propuesta se encuentra alineada con la arquitectura limpia promovida por el Scaffold Bancolombia. Cada nivel de pruebas valida una responsabilidad específica de la arquitectura, desde las reglas de negocio del dominio hasta los escenarios completos de fraude y autenticación ejecutados sobre la plataforma desplegada. Esto permite garantizar calidad, mantenibilidad y resiliencia en una solución crítica para la prevención de fraude financiero.
